Title: Create and Login

Primary Actor: User and Instructor

Stakeholders and Interests:

SuD: wants to create personal account and log into quiz system

Instructor: wants to login as the instructor usertype

Technician: authenticate student or instructor to log into the system

Precondition: SuD or Instructor got authentication, created own account

Main Scenario:

  1. 1.SuD uses email to create account and input the password
  2. 2.Instructor uses email to create account and input the password
  3. 3.SuD gets authentication successfully with inputting correct password
  4. 4.Instructor gets authentication successfully with inputting correct password
  5. 5.All users use email as login name and input password to log into the quiz

Extensions:

1a. SuDs create their own account including their email and password.

2a. Instructors create their own account including their email and password.

3. The SuD confirms login and displays according access that the account is allowed
