Title: Create and Login

Primary Actor: User and Instructor

Stakeholders and Interests:

SuD: wants to create personal account and log into quiz system inorder to take quizzes

Instructor: wants to create personal account and log into quiz system inorder to create quizzes

Technician: authenticate student or instructor to log into the system

Precondition: SuD or Instructor got authentication, created own account

Main Scenario:

  1. 1.SuD type the username to create account and set the password
  2. 2.Instructor type the username to create account and set the password
  3. 3.SuD gets authentication successfully with inputting correct password
  4. 4.Instructor gets authentication successfully with inputting correct password
  5. 5.After creating account successfully, data will be stored into persist

Extensions:

1a. SuDs can only create account under student pages

2a. Instructors can only create account under instructor pages

3a. All users are able to log in after create own accounts

3. The SuD confirms login and displays according access that the account is allowed
